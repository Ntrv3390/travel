## Key Features Implemented

### ✅ Pricing Consistency
**Critical requirement**: All 3 locations have same price
- Feed JSON: Generated by `feed_generator.go`
- JSON-LD: Generated by `jsonld_builder.go`
- Checkout: Calculated by `pricing/engine.go`

All call the **same PricingEngine** function to ensure consistency.

### ✅ Feed Sharding
- Automatically splits feeds > 500 products
- All shards share same nonce for atomic upload
- Supports inventories of any size

### ✅ Error Tracking
- All uploads logged to `google_feed_status` table
- Error messages recorded for debugging
- Status dashboard available via API

### ✅ Security
- SSH key authentication (Ed25519)
- Docker secrets for key storage
- Git protection (.gitignore configured)
- No credentials in code

### ✅ Automation
- Daily cron job (configurable)
- Can be manually triggered
- Runs in background without blocking

## Data Flow

```
HEADOUT API
    ↓
PostgreSQL (experiences + pricing_rules)
    ↓
    ├→ PricingEngine (calculates final prices)
    │
    ├→ FeedGenerator (builds GTTD JSON)
    │   └→ Shards if needed
    │   └→ Writes to /tmp/gttd_feeds/
    │
    ├→ JSONLDBuilder (builds schema.org)
    │   └→ Uses same PricingEngine
    │
    └→ SFTPUploader (SSH to Google)
           ↓
    Google Things to Do SFTP
           ↓
    Google Action Center
           ↓
    Google Search / Maps / Travel
```

## How to Deploy

### Phase 1: Credentials (Google)
1. Submit interest form
2. Sign CLA
3. Receive SFTP credentials

### Phase 2: Setup (Operations)
```bash
# 1. Generate SSH keys
cd secrets/
ssh-keygen -t ed25519 -C "traviia-gttd-sftp" -f gttd_dev_ssh_key -N ""
ssh-keygen -t ed25519 -C "traviia-gttd-sftp-prod" -f gttd_prod_ssh_key -N ""

# 2. Send public keys to Google
cat gttd_dev_ssh_key.pub
cat gttd_prod_ssh_key.pub

# 3. Update .env with Google-provided credentials
# 4. Start application
docker-compose up -d
```

### Phase 3: Population (Database)
```sql
-- Map locations
INSERT INTO poi_mappings (headout_city, google_place_id, is_verified)
VALUES ('New York', 'ChIJOwg_06VPwokR4v4GfH-j67w', true);

-- Enable experiences
UPDATE experiences SET gttd_enabled = true WHERE poi_id IS NOT NULL;
```

### Phase 4: Testing (Validation)
```bash
# Trigger upload
curl -X POST http://localhost:8080/api/v1/gttd/trigger-upload \
  -H "Content-Type: application/json" \
  -d '{"env":"dev"}'

# Check status
curl http://localhost:8080/api/v1/gttd/status?env=dev

# Verify in Google Action Center
```

### Phase 5: Go Live (Production)
- Update GTTD_ENV=production
- Verify in production SFTP
- Monitor daily uploads
- Check Action Center
